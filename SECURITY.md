# Security Policy

Thank you for helping keep API Sentinel and its users safe.

## Reporting a Security Issue

Please do not report security vulnerabilities in public GitHub issues.

Instead, send a private report to:

- knit16643@gmail.com

When possible, include:

- a clear description of the issue
- affected version(s)
- reproduction steps
- proof-of-concept details, if safe to share
- potential impact
- any suggested mitigation

## Response Expectations

We aim to:

- acknowledge receipt of a security report promptly
- investigate and validate the issue
- coordinate remediation and release timing
- credit researchers when appropriate and agreed

Response times may vary depending on severity, complexity, and release timing.

## Product Security Notes

- API Sentinel is distributed as compiled software. The source code is private and is not published in this repository.
- API Sentinel is designed for local-first workflows so API requests, collections, environments, and related data can remain on the local machine.
- No forced login is required to use the application.
- No telemetry is enabled by default unless explicitly documented for a specific release.

If a future release introduces behavior that changes security, network, storage, or telemetry expectations, that change should be documented in release notes and product documentation.

## Supported Versions

Security fixes are typically provided for the latest public release. When reporting an issue, please mention the exact version you tested.
