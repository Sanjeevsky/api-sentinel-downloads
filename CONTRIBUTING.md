# Contributing

Thanks for your interest in API Sentinel.

This repository is a public download and documentation repository. The application source code is private, so external code contributions are not accepted here.

## What You Can Contribute

You can still help improve the product by opening issues for:

- bug reports
- feature requests
- usability feedback
- installation problems
- documentation improvements

## Issue Guidelines

When opening an issue, please include:

- your operating system and version
- the API Sentinel version you are using
- clear reproduction steps
- expected behavior
- actual behavior
- screenshots or logs, when helpful

## Before Opening an Issue

- check whether the issue already exists
- verify you are using the latest public release
- include enough detail for the team to reproduce the problem

## Security Issues

Please do not open public issues for security vulnerabilities.

Use the process in [SECURITY.md](./SECURITY.md) instead.

## Feature Requests

If you are evaluating API Sentinel as an API client, REST client, or API testing tool, feature requests that explain your workflow are especially helpful. Concrete examples make prioritization easier.
